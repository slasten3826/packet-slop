# Lua L2 Boundary Stand

Это первый живой стенд второго слоя.

Он не доказывает канон `L2`.
Он делает только одно:

- превращает текущую гипотезу о `L2` в исполнимое тело

Здесь `L2` собран как:

- `hex`-тор
- 4 типа узлов
  - `OBSERVE`
  - `CHOOSE`
  - `ENCODE`
  - `RUNTIME`
- 4 ходока
- синтетическое давление снизу (`L1-like chaos pressure`)
- верхний `calm reservoir` как грубый суррогат `L3`

## Что делает стенд

На каждом тике:

1. `OBSERVE`-узлы читают synthetic chaos pressure
2. давление диффундирует по локальным hex-связям
3. ходоки двигаются по сетке
4. `CHOOSE` схлопывает маршрут
5. `ENCODE` превращает устойчивый ход в crystal candidate
6. `RUNTIME` принимает crystal candidate и поднимает `calm`

Это не полная packet-машина.
Это boundary sandbox.

## Запуск

```bash
lua main.lua
lua main.lua 12 10 128 12345
```

Аргументы:

1. `width`
2. `height`
3. `ticks`
4. `seed`

## Что печатается

- карта типов узлов
- checkpoint-и по тикам
- число живых ходоков
- суммарный charge
- число collapse events
- число encode events
- уровень calm reservoir

## Как читать результат

Если стенд работает осмысленно, то должно быть видно:

- `OBSERVE` действительно поднимает давление из нижнего контура
- `CHOOSE` не даёт всему расползтись равномерно
- `ENCODE` начинает срабатывать только на устойчивых маршрутах
- `RUNTIME` накапливает не сырой шум, а уже прошедший через boundary crystal

Это ещё не ответ про истинный `L2`.
Но это уже первый способ пощупать его руками.
