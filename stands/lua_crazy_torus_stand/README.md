# Lua Crazy Torus Stand

Это первый рабочий sandbox для `L1 chaos`.

Он не является нейросетью целиком.
Он не является `packet`.
Он не пытается говорить.

Его задача одна:

- проверить, может ли baseline `crazy-torus` жить как первый слой

## Что внутри

- `main.lua`

Текущая реализация:

- `1D ring`
- общий `core`
- `core + trace + phase`
- seed-grown field
- варианты `A`, `B`, `C`, `T3`, `T10`

## Запуск

```bash
lua main.lua 32 128 12345 A
lua main.lua 64 256 12345 B
lua main.lua 128 1024 12345 C
lua main.lua 128 1024 12345 T3
lua main.lua 128 1024 12345 T10
```

Аргументы:

1. размер кольца
2. число тиков
3. seed
4. вариант (`A`, `B`, `C`, `T3`, `T10`)

## Что печатается

- checkpoint-и по тикам
- текущая позиция
- carry-state
- fingerprint
- trace density
- distinct core count
- distinct trace count
- preview `core`
- preview `trace`

## Текущий статус

`Variant A`:

- запускается
- оставляет trace
- быстро уходит в короткие повторяющиеся циклы

`Variant B`:

- тоже запускается
- не разваливается
- даёт более богатое `core`
- выглядит живее baseline

`Variant C`:

- тоже запускается
- не разваливается
- даёт заметно более богатую внутреннюю форму
- по матрице `ring/seed` стабильно превосходит `B` по `distinct_core`
- на stress-test растёт вместе с размером кольца

`Variant T3`:

- запускается
- не разваливается
- на первых сравнениях превосходит одиночный `C`
- выглядит как первый рабочий тройной trace-режим

`Variant T10`:

- запускается
- не разваливается
- на первых сравнениях превосходит `T3`
- выглядит как первый рабочий режим сильного многотрейсного давления

Это ещё не победа.
Но это уже первый осмысленный шаг дальше baseline.

Первые stress-test числа для `Variant C`:

- `ring=32 -> distinct_core ~ 17-20`
- `ring=64 -> distinct_core ~ 28-31`
- `ring=128 -> distinct_core ~ 55-65`
- `ring=256 -> distinct_core ~ 115-137`

Это первый сильный признак,
что `chaos-core` масштабируется через размер тора,
а не просто повторяет короткий цикл на большем поле.

Длинные прогоны для `Variant C`:

- `128 / 4096 -> distinct_core=53`
- `256 / 8192 -> distinct_core=113`
- `512 / 8192 -> distinct_core=169`

Это показывает:

- режим не схлопывается на длинной дистанции
- богатство поля удерживается
- рост кольца продолжает давать рост внутренней формы

Первые сравнения `C` vs `T3`:

- `64 / 512 -> C=29, T3=43`
- `128 / 1024 -> C=55, T3=61`
- `256 / 2048 -> C=115, T3=136`

То есть тройной режим пока не ломает слой,
а делает его богаче.

Первые сравнения `C` vs `T3` vs `T10`:

- `64 / 512 -> C=29, T3=43, T10=45`
- `128 / 1024 -> C=55, T3=61, T10=71`
- `256 / 2048 -> C=115, T3=136, T10=158`

Это значит:

- `T10` пока тоже не ломает слой
- и на первых прогонах делает `core` ещё богаче, чем `T3`
- но его ещё надо мучить длинными прогонами, чтобы не принять шум за победу

На текущий момент:

- `C` — сильный одиночный baseline
- `T3` — первый удачный многотрейсный режим
- `T10` — текущий грязный лидер по richness
