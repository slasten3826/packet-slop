# Lua Eva.00 L2-L3 Stand

Это первый связанный стенд `Eva.00` для двух верхних тел машины:

- `L2` как граф формирования формы
- `L3` как life-space, где формы живут, пока не сдохнут

Стенд не является каноном.
Он нужен, чтобы проверить первую цельную физику `L2 -> L3`.

## Что здесь считается `Eva.00`

### L2

`L2` здесь не цепочка и не 4 узла.

`L2` — это hex-graph из произвольного количества узлов, но только 4 типов:

- `OBSERVE`
- `CHOOSE`
- `ENCODE`
- `RUNTIME`

То есть онтология `L2` ограничена четырьмя сущностями,
но тело `L2` может быть большим.

### L3

`L3` здесь не VM и не полный trigram-canon.

Это первый `Eva.00` life-space с четырьмя режимами жизни формы:

- `RUNTIME`
- `CYCLE`
- `LOGIC`
- `MANIFEST`

Идея такая:

- `OBSERVE` касается `L1 <-> L2`
- `RUNTIME` касается `L2 <-> L3`
- `MANIFEST` касается `L3 <-> L4`

Поэтому в `Eva.00` мы сознательно не тянем `OBSERVE` внутрь `L3`.

## Как работает стенд

1. Из `L1 ring/CW` выводится размер `L2`.
2. `L2` строится как большая hex-сетка из 4 типов узлов.
3. Давление снизу поднимает локальную активацию `L2`.
4. Стабильные `ENCODE`-узлы рождают `L3`-кристаллы.
5. Кристалл несёт:
   - `mode`
   - `energy`
   - `pu_stock`
   - `target`
   - `span`
6. `L3` принимает эти кристаллы в своё life-space.
7. Пока у кристалла есть `pu_stock`, он влияет на `L3`.
8. Когда `pu_stock` заканчивается, кристалл умирает и уходит в exhausted pool.

## Как здесь выбирается mode кристалла

Это временная `Eva.00`-эвристика:

- устойчивый и плотный узел чаще даёт `RUNTIME`
- конфликтная локальная среда чаще даёт `LOGIC`
- колебательная локальная среда чаще даёт `CYCLE`
- сильный верхний выход чаще даёт `MANIFEST`

Это не канон `PL`.
Это первый характер `Eva.00`, который можно крутить и сравнивать.

## Запуск

```bash
lua main.lua
lua main.lua 4096 256 128 12345
```

Аргументы:

1. `l1_ring`
2. `l1_cw`
3. `ticks`
4. `seed`

## Что смотреть в выводе

- размер `L2`
- `PU_max`
- сколько кристаллов породил `L2`
- какие mode кристаллов появились
- сколько кристаллов ещё живо в `L3`
- сколько уже exhausted
- `L3 readout`
- `manifest`

## Что этот стенд должен проверить

- может ли `L2` жить как большой 4-type graph
- может ли `L2` не просто гореть, а реально рождать формы для `L3`
- может ли `L3` жить как место жизни формы, а не как бесплатная ванна
- есть ли у `Eva.00` уже узнаваемый характер до всякого канона
