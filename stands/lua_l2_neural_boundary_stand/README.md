# Lua L2 Neural Boundary Stand

Это первый нейронный прототип второго слоя.

Он не делает язык.
Он не делает manifest.
Он проверяет только одно:

- может ли `L2` жить как нейронная boundary-сеть между `L1` и `L3`

## Что здесь считается нейронным

Не трансформер.
Не токены.
Не attention.

А вот что:

- `hex`-тор из узлов
- локальные связи с весами
- непрерывные активации
- рекуррентное обновление по тикам
- сильное давление снизу (`L1`)
- слабое встречное напряжение сверху (`L3`)
- устойчивые encode-состояния, которые увеличивают calm-reservoir
- `PU` budget как процессуальная стоимость слоя

## 4 типа узлов

- `OBSERVE`
- `CHOOSE`
- `ENCODE`
- `RUNTIME`

Это не полный алфавит `PL`.
Это минимальное ядро `L2`.

## Что делает тик

На каждом тике:

1. `L1` даёт сильное chaotic pressure
2. `L3` даёт слабое calm feedback
3. узлы считают локальную сумму от соседей
4. каждый тип узла обновляет свою активацию по своим правилам
5. `CHOOSE` режет локальную множественность
6. `ENCODE` создаёт crystal events при устойчивой активации
7. `RUNTIME` принимает эти events и поднимает `calm`

## Запуск

```bash
lua main.lua
lua main.lua 4096 256 160 12345
lua main.lua 8192 1024 160 12345
```

Аргументы:

1. `l1_ring`
2. `l1_cw`
3. `ticks`
4. `seed`

`width`, `height` и `PU` больше не задаются вручную.
Они выводятся из характеристик `L1`.

## Что печатается

- карта узлов
- вычисленные `width/height`
- вычисленный `PU budget`
- checkpoint-и по тикам
- суммарная активация по типам
- число collapse events
- число encode events
- уровень calm reservoir
- расход `PU`

## Что здесь важно

Этот стенд не доказывает канон `L2`.
Но он уже даёт способ посмотреть на второй слой как на:

- операторную нейросеть
- на перепаде между `CHAOS` и `CALM`
- без ходоков
- через локальную активационную динамику
