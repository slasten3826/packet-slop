# stands

This directory contains executable research stands.

- `lua_crazy_torus_stand`
  CPU-transparent baseline for the L1 chaos law.
- `lua_l2_boundary_stand`
  first executable hex-boundary sandbox for the L2 hypothesis.
- `lua_l2_neural_boundary_stand`
  first neural hex-boundary sandbox for the L2 hypothesis.
- `lua_l3_substrate_stand`
  first recurrent substrate sandbox for the L3 hypothesis.
- `lua_eva00_l2_l3_stand`
  first connected Eva.00 sandbox for the L2->L3 body.
- `lua_l2_market_cost_stand`
  narrow economic sandbox for cheap chaos and expensive calm in L2.
- `lua_l2_l3_market_flow_stand`
  connected market sandbox for L2 economy and L3 form distribution.
- `lua_eva00_encode_core_stand`
  first autonomous CPU microcore for Eva.00 encode over L1 chaos.
- `lua_eva00_encode_core_l3_stand`
  connected CPU stand for Eva.00 encode core feeding the current L3 body.
- `lua_eva00_multi_l3_cost_stand`
  CPU stand for testing multiple L3 pockets with life-cost scaling by L3 count.
- `python_l4_to_l1_bootstrap_probe`
  first rough extractor of early transformer material for the `L4 -> L1` bridge.
- `python_l3_dynamic_lora_stand`
  first executable stand for `L3 -> L4` as a dynamic low-rank adapter over frozen transformer hidden states.
- `lua_l1_bootstrap_from_l4_stand`
  first rough `L1` stand where the ring is built directly from transformer bootstrap tokenization.
- `lua_l1_multiplier_stand`
  first rough `multi-L1` stand for testing one bootstrap world cloned into `M` parallel `L1` cores.
- `lua_l1_l2_multiplier_stand`
  first rough `Mx(L1+L2)` stand for testing one bootstrap world expanded into `M` parallel lower cores.
- `lua_l1_l2_l3_multiplier_stand`
  first rough `Mx(L1+L2) -> shared fat L3` stand for testing one bootstrap world with a single enlarged market.
- `lua_l1_l2_l3_transition_stand`
  first rough transition stand where `L2` has no `RUNTIME`, and `L3` begins through `encoded crystal + PU`.
- `lua_l1_l2_l3_emergent_field_stand`
  first rough stand where `L2` emits only seeds and `L3` modes emerge from field dynamics.
- `cuda_crazy_t3_stand`
  CUDA stand for the GPU body of the same law.
- `cuda_l2_neural_boundary_stand`
  first GPU test stand for the L2 neural boundary body.
- `cuda_l3_substrate_stand`
  first GPU test stand for the L3 substrate body.
- `cuda_eva00_l2_l3_stand`
  first GPU body for the connected Eva.00 L2->L3 stand.
- `cuda_eva00_encode_core_l3_stand`
  GPU body for the current Eva.00 encode-core -> L3 baseline.
- `cuda_eva00_multi_l3_cost_stand`
  GPU stand for testing multiple L3 pockets with life-cost scaling by L3 count.

These stands are not product code.
They exist to test transfer, parity, scaling, stress, and runtime asymmetry.
