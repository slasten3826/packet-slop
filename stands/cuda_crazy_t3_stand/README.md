# CUDA Crazy T3 Stand

Это первый GPU-порт для `L1 chaos`.

Он не пытается улучшать `T3`.
Он не пытается переизобретать `L1`.

Его задача одна:

- тупо и буквально перенести `T3` с `Lua` на `CUDA`

## Статус

Сейчас это уже живой `.cu` reference-port.

Он нужен, чтобы:

- удержать parity с Lua-эталоном
- зафиксировать layout
- не расползтись по архитектуре раньше времени
- только потом идти в настоящий CUDA-kernel path

## Что должно совпадать с Lua

- `crazy`
- seeded ring
- `phase`
- `3` trace
- `3` carry-state
- `3` позиции
- checkpoint-метрики

## Важно

Если для удобства GPU захочется:

- переписать физику шага
- упростить trace
- убрать self-overwriting
- подменить `crazy`

значит порт уже испорчен.

## Уже подтверждено

После исправления первой инициализации `carry`
`.cu`-версия совпала с Lua `T3` на контрольных прогонах:

- `64 / 512 / 12345`
- `128 / 1024 / 12345`

Совпали:

- `carry`
- `fingerprint`
- `trace_density`
- `distinct_core`
- `distinct_trace`
- `core_preview`

После этого реальный `use_gpu=1`
тоже был прогнан на хостовой `GTX 1080`.

Контрольный прогон:

```bash
./crazy_t3 64 512 12345 3 0 1
```

Что подтвердилось:

- GPU-путь реально жив
- checkpoint-метрики совпали с `Lua T3`
- `L1 law` выдержал перенос на настоящее устройство

## Benchmark mode

Стенд теперь умеет:

- менять `trace_count`
- мерить `elapsed_us`
- работать в quiet CSV-режиме

Примеры:

```bash
./crazy_t3 64 512 12345 3
./crazy_t3 64 512 12345 10
./crazy_t3 256 2048 12345 32 1
./crazy_t3 64 512 12345 3 0 1
./benchmark.sh 12345
./benchmark.sh 12345 1
```

Аргументы:

1. `ring_size`
2. `ticks`
3. `seed`
4. `trace_count`
5. `quiet` (`1` = одна CSV-строка)
6. `use_gpu` (`1` = реальный CUDA path)

## Сборка

Сейчас стенд по умолчанию собирается так:

- `nvcc` из `/opt/cuda/bin`
- host compiler = `/usr/bin/g++-14`

Это важно для `cuda-pascal` на `GTX 1080`.

Обычный путь:

```bash
make
./crazy_t3 64 512 12345 3 0 1
```
