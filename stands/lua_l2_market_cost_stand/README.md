# Lua L2 Market Cost Stand

Это узкий экономический стенд для `L2`.

Он не проверяет топологию `L2`.
Он проверяет только одну гипотезу:

- `CHAOS` в `OBSERVE` и раннем `CHOOSE` дешёвый
- `ENCODE` резко удорожает всё после себя
- `CALM` в `OBSERVE / CHOOSE / RUNTIME` дорогой

## Идея

Внутри стенда рождаются сырые идеи из `CHAOS`.

Дальше у идеи есть два маршрута:

1. короткий сырой путь  
   `OBSERVE(raw) -> CHOOSE(raw) -> RUNTIME(raw)`

2. длинный дорогой путь  
   `OBSERVE(raw) -> CHOOSE(raw) -> ENCODE -> OBSERVE(calm) -> CHOOSE(calm) -> RUNTIME(calm)`

Первый путь:

- дешёвый
- быстрый
- даёт слабый runtime artifact

Второй путь:

- дорогой
- медленный
- даёт более качественный runtime artifact

## Что проверяется

- сколько идей вообще проходит через слой
- сколько денег сжигается на каждом операторе
- сколько форм доходит до runtime сырым путём
- сколько форм доходит до runtime calm-путём
- насколько encode действительно удорожает всё после себя

## Запуск

```bash
lua main.lua
lua main.lua 2000 12345
```

Аргументы:

1. `ideas`
2. `seed`

## Что печатается

- число сырых идей
- сколько умерло на раннем `CHOOSE`
- сколько прошло по raw-path
- сколько прошло по calm-path
- среднее качество runtime artifact по обоим путям
- расход по операторам
- общий расход

## Что это не делает

- не доказывает канон `L2`
- не моделирует hex-graph
- не заменяет `Eva.00`

Это только экономический микростенд, чтобы пощупать закон:

> `ENCODE` не просто дорогой сам по себе,  
> он делает всё после себя дорогим.
