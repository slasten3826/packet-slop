# Lua Eva.00 Encode Core Stand

Это первый автономный CPU-стенд для `Eva.00 ENCODE core`.

Он не моделирует весь `L2`.
Он моделирует только его живое ядро:

- `OBSERVE`
- `CHOOSE`
- `ENCODE`
- `RUNTIME`

над уже готовым `L1 chaos`.

## Главная идея

Это не последовательная программа.

Это набор параллельных процессов `EncodeProcess`,
каждый из которых живёт поверх `L1` и каждый тик:

- держит hidden `CONNECT`
- тянет `CHAOS`
- пассивно/реактивно включает `OBSERVE`
- пассивно/реактивно включает `CHOOSE`
- постепенно собирает `CALM`
- и иногда выпускает crystal в `RUNTIME`

## Что здесь важно

- `L1` считается уже готовым и используется как внешний источник хаоса
- `L3` тут вообще не влияет на `L2`
- `RUNTIME` только принимает готовое
- `OBSERVE` и `CHOOSE` имеют доступ к `L1`
- hidden `CONNECT` у `ENCODE` тикает всегда

## Что печатается

- сколько процессов стартовало
- сколько живо к концу
- сколько умерло неуспешно
- сколько успешно выпустило crystal
- сколько раз реально вызывались:
  - `OBSERVE(raw)`
  - `OBSERVE(calm)`
  - `CHOOSE(raw)`
  - `CHOOSE(calm)`
- среднее время жизни
- средняя coherence
- средний burn

## Запуск

```bash
lua main.lua
lua main.lua 128 256 12345
```

Аргументы:

1. `ticks`
2. `process_count`
3. `seed`
