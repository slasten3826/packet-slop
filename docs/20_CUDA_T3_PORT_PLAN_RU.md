# 20. CUDA T3 Port Plan

Этот документ фиксирует первый практический план GPU-порта.

Не всей машины.
Не всего `L1`.
Только:

- текущий рабочий `T3`
- как буквальный перенос на `CUDA`

## Цель

Сделать минимальный GPU-стенд, который:

- повторяет физику `T3`
- не вводит новых архитектурных сущностей
- считает то же самое, что Lua-эталон
- позволяет сравнивать `CPU` и `GPU` один в один

## Что переносим

Переносится только:

- `1D ring`
- общий `core`
- общий `phase`
- `3` trace
- `3` carry-state
- `3` позиции
- `crazy`
- шаг `T3`

Не переносится:

- `T10`
- обучение
- верхние слои
- язык
- никакая новая оптимизация

## Архитектурный принцип

`Lua` остаётся reference.

`CUDA` — только acceleration branch.

Если `CUDA`-ветка требует менять саму физику `T3`,
значит она уже не годится как первая проверка эталона.

## Минимальная структура проекта

Каталог:

- `packet/cuda_crazy_t3_stand`

Файлы:

- `README.md`
- `main.cu`
- `Makefile`

## Минимальный режим v0

Первая версия должна уметь:

1. создать seeded ring
2. поднять `T3`-состояние
3. прогнать `N` тиков
4. снять checkpoints
5. вывести:
   - `fingerprint`
   - `trace_density`
   - `distinct_core`
   - `distinct_trace`

## Правило v0

На первом шаге допускается:

- тупой GPU-код
- некрасивая memory layout
- неидеальная производительность

Не допускается:

- изменение логики `crazy`
- изменение логики `T3`
- подмена эталона ради удобства GPU

## Первый критерий успеха

Для одинаковых:

- `ring`
- `ticks`
- `seed`

Lua и CUDA дают:

- тот же тип checkpoint-динамики
- близкий `distinct_core`
- близкий `distinct_trace`
- тот же общий рисунок fingerprints

## Первый критерий смерти

Если GPU-порт требует:

- перепридумать шаг
- перепридумать trace
- перепридумать carry
- ломать reference ради kernel convenience

ветка убивается.
