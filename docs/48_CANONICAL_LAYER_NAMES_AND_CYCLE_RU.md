# Canonical Layer Names And Cycle

This is the current canonical naming of the packet machine stack.

## Canonical names

| Layer | Canonical name | Short meaning |
|---|---|---|
| `L1` | `CHAOS` | raw potentiality, emission, underdefined source |
| `L2` | `FORMATION` | formation of form, production, shaping machinery |
| `L3` | `MARKET` | life-space of forms, competition, holding, burn |
| `L4` | `DEMAND` | buyer, demand, I/O, manifestation, fate of forms |

## Canonical cycle

```txt
CHAOS (L1) → FORMATION (L2) → MARKET (L3) → DEMAND (L4)
     ↑                                            |
     └──────── composting residue ◄───────────────┘
```

## Reading

- `CHAOS`
  raw source of potentiality and entropy.
- `FORMATION`
  where chaos is observed, narrowed, encoded, and prepared as form.
- `MARKET`
  where forms live, compete, persist, burn, and possibly accumulate as memory-like state.
- `DEMAND`
  where external or internal demand selects from the market, manifests outward, and sends residue back into chaos.

## Why this naming is good

- It matches the current economic reading of the machine.
- It matches the already discovered `L1 -> L2 -> L3 -> L4 -> L1` cycle.
- It leaves room for zig `TENSION` logic inside `DEMAND`.
- It makes `L4` readable both as user I/O and as the upper pressure that closes the loop.

## Current interpretation of `L4`

`DEMAND` does not replace zig `TENSION`.

It gives it a clearer role:

- `TENSION` is the internal law of `L4`
- `DEMAND` is the machine-facing and user-facing meaning of `L4`

So the current reading is:

- `L4 internal law` = `TENSION`
- `L4 canonical role` = `DEMAND`

## Status

This naming is accepted as current canon unless later research breaks it.
