# L1 Packet Neuroprototype Technical Spec

Этот документ фиксирует `L1` как технический нейропрототип.

Не как сказку.
Не как философское введение.
Не как россыпь стендов.

А как первую реальную машинную форму,
которую можно:

- читать
- собирать
- сравнивать
- масштабировать
- вести в двух ветках исполнения

## 1. Что такое `L1`

`L1` — это первый живой слой `packet-neural`.

Его роль:

- не хранить готовую память
- не кодировать гладкий embedding
- не быть decoder/encoder

Его роль такая:

- быть живой `chaos-medium`
- удерживать поле в состоянии локальной нелинейной жизни
- давать только отпечатки и динамику,
  а не готовую кристаллическую форму

Текущая каноническая физика `L1`:

- `crazy-torus / T3`

## 2. Инварианты слоя

`L1` считается настоящим только если сохраняет следующие свойства.

### 2.1 Seed-grown field

Поле не задаётся полностью снаружи.
Оно разворачивается из малого начала.

### 2.2 Общее поле

Код и данные не разделены жёстко.
След исполнения и есть часть новой программы поля.

### 2.3 Локальная нелинейная операция

Базовая физика слоя строится вокруг `crazy`,
а не вокруг линейной агрегации признаков.

### 2.4 Самоизменение при тике

Каждый тик меняет сам субстрат.
Чтение и исполнение не нейтральны.

### 2.5 Необратимость

Преобразование не должно быть полностью обратимым.
Потеря формы встроена в саму физику.

### 2.6 Позиционно-зависимый смысл

Одинаковые значения в разных местах поля
не обязаны означать одно и то же.

### 2.7 Отсутствие собственной кристаллизации

`L1` не должен сам становиться памятью или calm-state.
Если в нём возникает стабильная готовая сущность,
значит слой ушёл не туда.

## 3. Состав состояния

Текущий минимальный `L1`-объект:

- `1D ring`
- `core`
- `trace`
- `phase`
- `carry-state A`
- `active position p`

### 3.1 `core[i]`

Основное поле узла.

Смысл:

- живая текущая ткань слоя
- не память
- не архив
- не latent codebook

### 3.2 `trace[i]`

Активный след исполнения.

Смысл:

- хвост предыдущих проходов
- давление прошлого на следующий тик
- переписываемая исполнительная память

### 3.3 `phase[i]`

Малый позиционный модификатор.

Смысл:

- не дать состоянию быть полностью автономным
- встроить зависимость смысла от места

### 3.4 `A`

Carry-state текущего активного trace.

Смысл:

- живой перенос внутреннего напряжения через последовательность тиков

### 3.5 `p`

Текущая активная позиция.

Смысл:

- локализация фронта исполнения

## 4. Tick model

Базовый tick `L1` строится так:

1. выбрать текущий узел `p`
2. выбрать сосед или следующий рабочий узел `q`
3. собрать локальный operand
4. применить `crazy(A, operand)`
5. обновить `A`
6. мутировать `core[p]`
7. обновить `trace[p]`
8. передвинуть фронт исполнения

Это не нейронная активация.
Это минимальная физика хаотического слоя.

## 5. Что делает каждая функция слоя

### 5.1 Seed

Функция:

- запустить среду из малого начала
- не дать полю быть полностью внешне расписанным заранее

### 5.2 `crazy`

Функция:

- реализовать встроенный `ENCODE + DISSOLVE`
- локально изменить поле необратимым способом

### 5.3 Trace update

Функция:

- переносить давление предыдущих проходов
- делать второй и третий круг не равными первому

### 5.4 Phase / positional modifier

Функция:

- встроить локальную контекстность
- сломать иллюзию универсального значения вне позиции

### 5.5 Fingerprint read

Функция:

- наблюдать слой, не пытаясь снять с него полную устойчивую форму
- получать сигнал жизни, а не полную расшифровку сущности

## 6. Метрики жизни

Минимальный набор метрик для `L1`:

- `fingerprint`
- `trace_density`
- `cycle_delta`
- `distinct_core_count`
- `distinct_trace_count`
- `elapsed_us`

### 6.1 `fingerprint`

Короткий отпечаток поля.

Нужен чтобы:

- сравнивать runs
- проверять parity
- видеть стагнацию / изменение

### 6.2 `trace_density`

Нужен чтобы:

- видеть, насколько живой след реально распределён по полю

### 6.3 `cycle_delta`

Нужен чтобы:

- увидеть, меняется ли следующий проход под давлением уже оставленного следа

### 6.4 `distinct_core_count`

Нужен чтобы:

- измерять богатство поля
- видеть плато и перегибы насыщения

### 6.5 `distinct_trace_count`

Нужен чтобы:

- видеть разнообразие самого активного следа, а не только core

### 6.6 `elapsed_us`

Нужен чтобы:

- измерять вычислительную цену выбранного режима

## 7. Ручки слоя

У `L1` уже найдены две независимые ручки.

### 7.1 `ring`

Смысл:

- ёмкость среды
- размер поля
- пространство для циркуляции и насыщения

### 7.2 `trace_count`

Смысл:

- давление жизни на среду
- число параллельных активных проходов / следов

Важно:

- `ring` и `trace_count` не одно и то же
- они увеличивают разные аспекты мощности слоя

Текущая терминологическая замена:

- `trace_count` = `CW`
- `CW` = `crazy waves`

Отдельная фиксация асимметрии:

- [26_RING_CW_ASYMMETRY_RU.md](26_RING_CW_ASYMMETRY_RU.md)

## 8. Scaling

### 8.1 CPU scaling

На `CPU` картина пока выглядит так:

- рост `trace_count` почти прямо повышает цену
- рост `ring` повышает ёмкость среды
- `T3` остаётся хорошим baseline-режимом

То есть `CPU L1`:

- хорошо читается
- хорошо дебажится
- годится как эталон

### 8.2 GPU scaling

На `GPU` картина уже другая:

- стоимость на малых кольцах местами почти плоская
- на больших кольцах рост `trace_count` не обязан линейно удорожать прогон
- видны резонансные числа:
  - `32`
  - `64`
  - `128`
  - `256`
  - `512`
  - `1024`

То есть `GPU L1`:

- живёт по другой экономике
- чувствителен к порогам железа
- не должен проектироваться как просто ускоренный CPU

## 9. Две параллельные ветки

С этого момента `L1` официально ведётся в двух ветках.

### 9.1 `CPU branch`

Назначение:

- эталон
- прозрачность
- дебаг
- малые и средние стенды
- baseline для сравнения законов

### 9.2 `GPU branch`

Назначение:

- большие поля
- pressure regimes
- другая упаковка тиков
- поиск машинно-резонансных чисел

Главная оговорка:

- `GPU branch` не обязана повторять форму `CPU branch`
- у них один закон слоя
- но могут быть разные вычислительные тела

## 11. Жёсткий вывод

`L1` уже надо понимать не как гипотезу,
а как технический packet-neuroprototype.

На текущий момент уже доказано:

- слой жив
- слой переносим
- слой измерим
- слой имеет собственные ручки мощности
- слой уже распадается на `CPU/GPU` ветки

Это и есть первая настоящая машинная форма `packet-neural`.
