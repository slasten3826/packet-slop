# Ring/CW Asymmetry

Этот документ фиксирует важный результат `L1`.

Мы отдельно проверили асимметрию:

- маленький `ring` + большой `CW`
- большой `ring` + маленький `CW`

и сделали это и на `CPU`, и на `GPU`.

## Термины

- `ring` = размер тора / размер поля / ёмкость среды
- `CW` = `crazy waves` / число `crazy`-волн / давление жизни на среду

Это две разные ручки.
Они не взаимозаменяемы.

## Главный результат

`ring` и `CW` делают разное:

- `ring` даёт пространство
- `CW` даёт давление

Из этого следует:

- маленький `ring` нельзя "спасти" бесконечным `CW`
- большой `ring` нельзя полностью заменить малым `CW`
- зрелый режим слоя требует и пространства, и давления,
  но разная вычислительная среда любит разные пропорции

## CPU physics

На `CPU` картина такая:

- большой `ring` даёт богатство относительно дёшево
- высокий `CW` даёт богатство дорого
- маленький `ring` + большой `CW` быстро уходит в перегрев

### Показательные числа

#### Малый `ring`, высокий `CW`

- `256 / CW3   -> 3148 us,    distinct_core=149`
- `256 / CW64  -> 68066 us,   distinct_core=190`
- `256 / CW256 -> 269316 us,  distinct_core=187`

#### Большой `ring`, малый `CW`

- `1024 / CW3 -> 12852 us, distinct_core=474`
- `2048 / CW3 -> 25459 us, distinct_core=928`
- `4096 / CW3 -> 50963 us, distinct_core=1691`
- `8192 / CW3 -> 103162 us, distinct_core=2924`

### Вывод по CPU

`CPU` любит:

- широкий мир
- малое число волн
- sparse-chaos physics

`CPU` не любит:

- тесное поле
- чрезмерное число волн
- overpressured small field

То есть:

- `CPU` хорошо растит среду
- но плохо переносит сверхплотное давление жизни внутри маленького мира

## GPU physics

На `GPU` картина другая:

- высокий `CW` не обязан удорожать режим
- иногда высокий `CW` даже делает режим выгоднее
- большой `ring` и высокий `CW` могут жить вместе естественно

### Показательные числа

#### Малый `ring`, высокий `CW`

- `256 / CW3   -> 80860 us, distinct_core=149`
- `256 / CW64  -> 73630 us, distinct_core=190`
- `256 / CW256 -> 62485 us, distinct_core=187`

#### Большой `ring`, малый `CW`

- `1024 / CW3 -> 281021 us, distinct_core=474`
- `2048 / CW3 -> 583958 us, distinct_core=928`
- `4096 / CW3 -> 1145620 us, distinct_core=1691`
- `8192 / CW3 -> 2393462 us, distinct_core=2924`

#### Большой `ring`, высокий `CW`

- `8192 / CW256  -> 2170791 us, distinct_core=4840`
- `8192 / CW2048 -> 2005095 us, distinct_core=5182`

### Вывод по GPU

`GPU` любит:

- плотный хаос
- большое число волн
- pressure-rich field

То есть:

- `GPU` хорошо переносит dense-chaos physics

## Неожиданный эффект: performance optimum и richness optimum

На `GPU` вылезло ещё одно важное свойство.

Есть:

- точка, где режим самый быстрый
- и точка, где режим самый богатый

И это не всегда одна и та же точка.

Например:

- `4096 / CW1024 -> distinct_core=3021`
- `4096 / CW4096 -> distinct_core=2661`, но быстрее

Это значит:

- больше волн не всегда значит "лучше"
- есть sweet spot
- после него давление может всё ещё нравиться железу,
  но уже выравнивать внутреннюю форму слоя

## Жёсткая формула

На текущем материале можно грубо зафиксировать:

- `CPU = wide sparse chaos`
- `GPU = dense pressured chaos`

Это не окончательная метафизика слоя.
Но это уже рабочая инженерная физика исполнения.

## Архитектурное следствие

Из этого прямо следует:

- `CPU branch` и `GPU branch` нельзя больше вести как одну и ту же инженерную форму

Наоборот:

- `CPU branch` должен быть оптимизирован под пространство и прозрачность
- `GPU branch` должен быть оптимизирован под давление и массовую локальную жизнь

## Практический итог

Когда дальше будем выбирать режимы `L1`,
нужно различать минимум 4 класса:

1. `compressed chaos`
   - маленький `ring`
   - высокий `CW`

2. `wide sparse chaos`
   - большой `ring`
   - малый `CW`

3. `balanced field`
   - большой `ring`
   - умеренный `CW`

4. `overpressured field`
   - очень большой `CW`
   - после точки оптимума

Это уже не просто benchmark-таблица.
Это карта режимов жизни слоя.
