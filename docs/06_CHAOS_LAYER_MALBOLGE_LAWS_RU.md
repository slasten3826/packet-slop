# Законы Chaos-Layer Через Malbolge

Этот документ фиксирует главное:

- `L1 chaos` нельзя проектировать как обычный latent layer
- его надо читать через `Malbolge`

Это не историческая справка.
Это рабочая спецификация инвариантов.

## Главное утверждение

Для `packet` `Malbolge` важен не как эзотерический язык,
а как ближайший известный субстрат,
в котором уже встроены нужные законы `chaos`.

Следовательно:

- будущий `chaos-layer` должен быть не просто графовым
- он должен быть `Malbolge-like`

## Что в Malbolge важно для chaos

### 1. DISSOLVE встроен в каждый тик

Главное свойство:

- инструкция не переживает своего исполнения
- после тика форма уже не та, что была до тика

Для `chaos-layer` это означает:

- слой должен самоизменяться при каждом шаге
- стабильный неизменяемый hidden state здесь недопустим

Если состояние читается и остаётся прежним,
это уже не тот chaos.

### 2. Код и данные живут в одном поле

В Malbolge нет чистого разделения:

- вот код
- вот данные

Они делят одно пространство.

Для `chaos-layer` это означает:

- нельзя жёстко делить управляющую форму и живое состояние
- поле должно быть общим
- вычисление и материал вычисления должны жить в одной среде

### 3. Смысл зависит от позиции

В Malbolge значение инструкции зависит не только от символа,
но и от его позиции.

Это означает:

- смысл рождается из отношения
- значение не самодостаточно

Для `chaos-layer` это означает:

- значение узла не должно быть полностью автономным
- интерпретация должна зависеть от положения / связи / контекста узла

### 4. ENCODE встроен в физику

`crazy` — не просто операция.
Это необратимое преобразование.

Для `chaos-layer` это означает:

- в самом слое должна быть встроенная необратимость
- потеря информации не должна быть только следствием внешнего сжатия
- без собственной необратимости слой станет слишком гладким

### 5. Seed порождает среду

Malbolge-память не просто заполняется нулями.
Она разворачивается из малого начала через правило.

Для `chaos-layer` это означает:

- вход должен быть семенем
- среда должна расти из семени
- полное поле не должно быть задано заранее как статический массив признаков

### 6. Кристаллизация внутри chaos невозможна

Если DISSOLVE встроен в каждый тик,
то сам `chaos` не может быть местом устойчивой кристаллизации.

Это означает:

- `chaos-layer` не должен сам становиться памятью
- кристалл должен рождаться только в `boundary -> calm`
- если внутри `chaos` появляются стабильные готовые сущности,
  это уже не тот слой

### 7. CHOOSE должен менять выбранное

В Malbolge переход пути не нейтрален.
Сам акт выбора модифицирует то, что было выбрано.

Для будущего `packet-native` пути это означает:

- `CHOOSE` не должен быть пассивным роутингом
- выбор должен влиять на сам путь

Это свойство позже пригодится для `boundary`,
но его корень уже лежит в физике хаоса.

## Что из этого следует для будущего chaos-layer

Минимальные инварианты `L1 chaos`:

- общее поле состояния
- seed-grown medium
- локальная нелинейная физика
- необратимость
- самоизменение при тике
- позиционно-зависимый смысл
- отсутствие собственной кристаллизации
- возможность снять только fingerprint, но не полную устойчивую форму

## Что нельзя делать

Нельзя делать:

- обычный latent vector
- smooth embedding field
- стандартный recurrent hidden state
- чистую память без саморазрушения формы
- слой, где состояние можно читать бесконечно без изменения

Если получается что-то из этого,
ветку надо убивать.

## Текущая рабочая формула

Не:

- graph chaos

А:

- `Malbolge-like graph chaos`

То есть граф сам по себе недостаточен.
Граф должен жить по malbolge-подобным законам.

## Практический вывод

Первый нейросетевой шаг нельзя строить так:

- "сделаем граф и обучим его"

Первый шаг должен звучать так:

- "сделаем минимальное графовое поле,
  которое obeys Malbolge-like laws"

Если это не получается,
нейросетевая ветка `packet` должна быть выброшена сразу.
