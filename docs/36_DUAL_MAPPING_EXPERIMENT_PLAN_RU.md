# Dual Mapping Experiment Plan

Этот документ фиксирует следующий инженерный шаг:

проверить гипотезу о двух внутренних маппингах машины через одинаковый цикл стендов на `CPU` и `GPU`.

## Коротко

Гипотеза:

- существует два разных внутренних маппинга
- они сильнее всего касаются `L2` и `L3`
- `L1` и `L4` остаются более инвариантными
- один маппинг, вероятно, естественно ляжет на `CPU`
- другой — на `GPU`

Пока это не доказано.

Поэтому сначала вводятся временные нейтральные имена.

## Временные имена

Первый маппинг:

- `Eva.00`

Второй маппинг:

- `Eva.01`

Важно:

эти имена пока не означают:

- `CPU`
- `GPU`
- `male`
- `female`

Они нужны только для честного этапа эксперимента без преждевременной догмы.

## Текущий рабочий порядок

1. Выбрать один маппинг как первый для проверки.
2. Реализовать его в `L2` и `L3`.
3. Прогнать один и тот же цикл тестов:
   - `CPU`
   - `GPU`
4. Зафиксировать:
   - устойчивость
   - richness
   - `PU`-расход
   - перегрев
   - поведение форм
5. Потом сделать то же самое со вторым маппингом.
6. Только после этого решать, есть ли естественная привязка:
   - `Eva.00 -> CPU`
   - `Eva.01 -> GPU`
   - или наоборот

## Стартовая договорённость

На ближайшем шаге:

- первый выбранный маппинг будет называться `Eva.00`
- второй выбранный маппинг будет называться `Eva.01`

Только если гипотеза подтвердится в стендах, можно будет потом переименовать их по факту, например:

- `Eva.CPU`
- `Eva.GPU`

или просто:

- `cpu`
- `gpu`

## Почему так

Это нужно, чтобы не сломать эксперимент преждевременной красивой интерпретацией.

Если заранее назначить:

- этот маппинг = `CPU`
- этот маппинг = `GPU`

то легко начать подгонять результаты под ожидание.

Временные имена удерживают инженерную честность.

## Что именно проверяется

Проверяется не только “какой маппинг красивее”.

Проверяется:

- как каждый маппинг формирует `L2`
- как каждый маппинг живёт в `L3`
- как он расходует `PU`
- как он ведёт себя на `CPU`
- как он ведёт себя на `GPU`

То есть это эксперимент не про названия, а про соответствие внутренней проекции конкретному вычислительному телу.

## Инвариант

Даже если маппинг меняется в `L2/L3`,
весь путь всё равно нужно тюнить целиком под конкретную ветку:

- `CPU + Eva.00`
- `GPU + Eva.00`
- `CPU + Eva.01`
- `GPU + Eva.01`

Потому что один и тот же маппинг в разных железных телах может давать разную жизнь слоя.

## Самая короткая формула

- сначала `Eva.00`
- потом `Eva.01`
- обе прогоняются на `CPU` и `GPU`
- только потом появляется право назвать их по настоящему
