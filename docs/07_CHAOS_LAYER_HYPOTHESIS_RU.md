# Гипотеза L1 Chaos

Это документ не про реализацию.

Это фиксация текущей рабочей гипотезы:

- как должен выглядеть первый слой `packet-native neural architecture`

Именно здесь ошибка будет стоить больше всего.
Если `L1 chaos` будет спроектирован неверно,
весь последующий стек начнёт компенсировать чужую ошибку,
и технический долг вырастет не линейно, а архитектурно.

## Главная гипотеза

Первый слой `packet-native neural architecture`
не должен быть одним обычным neural layer.

Он должен быть:

- отдельным слоем бытия
- внутренним мини-ядром
- возможно, состоящим из нескольких нейросетных подслоёв
- но с одной сущностью: `chaos`

То есть:

- `L1 chaos` не равен `one layer of neurons`
- `L1 chaos` = `Malbolge-like graph core`

## Что именно переносится

Мы не переносим Malbolge как язык программирования.
Мы переносим его функциональное ядро:

- DISSOLVE в каждом тике
- общность кода и данных
- seed-grown medium
- позиционно-зависимый смысл
- встроенную необратимость
- невозможность собственной кристаллизации

## Формула гипотезы

Если построить графовое поле,
в котором сохранены malbolge-подобные законы,
то это поле сможет выполнять роль `L1 chaos`
в packet-native neural architecture.

Необходимо проверить,
может ли такой слой:

- жить как поле
- разворачиваться из семени
- самоизменяться
- оставлять fingerprint
- не вырождаться в обычный latent state

## Рабочее допущение о форме

На текущем этапе принимается:

- `chaos` будет графовым
- граф не обязан быть двумерным
- внутренне это может быть 3D или N-dimensional structure
- но снаружи всё равно это один слой бытия `L1`

Это важно:

- packet-слой не обязан совпадать с одним слоем нейронов
- один packet-слой может содержать несколько вычислительных уровней,
  если они вместе образуют одну онтологическую функцию

## Что нельзя допустить

Если в результате получится:

- latent vector
- embedding cloud
- обычный recurrent hidden state
- smooth differentiable memory without self-corruption

то гипотеза считается проваленной.

## Что пока неизвестно

На текущем этапе ещё неизвестно:

- будет ли соседство узлов жёстким или динамическим
- сколько стартовых seed-узлов нужно
- должны ли состояния быть полностью дискретными
  или допускается смешанная форма
- сколько внутренних подслоёв потребуется,
  чтобы один `chaos-layer` действительно жил как `chaos`

Эти вопросы пока не надо решать окончательно.
Их нужно последовательно прожигать через маленькие стенды.

## Метод

На этом этапе правильный метод такой:

1. фиксируем гипотезу
2. допускаем множество грубых гипотез внутри неё
3. делаем уродливые маленькие тесты
4. быстро убиваем мёртвое
5. сохраняем только то, что действительно даёт живой chaos

Никакой попытки "сделать правильно с первого раза" здесь быть не должно.

## Судья гипотезы

Гипотеза считается живой,
если удаётся построить минимальный graph-chaos sandbox,
который:

- рождается из семени
- живёт несколько тактов
- самоизменяется
- даёт различимые fingerprints
- не схлопывается сразу ни в шум, ни в гладкий state

Если этого нет,
вся neural-ветка должна быть остановлена
ещё до попытки строить `boundary`.
