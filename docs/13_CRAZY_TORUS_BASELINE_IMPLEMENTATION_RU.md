# Crazy Torus Baseline Implementation

Этот документ фиксирует первую реализацию `L1 chaos`.

Выбранный вариант:

- `Variant A`
- буквальный baseline
- без обучения
- без нескольких trace-агентов
- без `boundary`, `calm`, `tension`

## Почему выбран именно он

Потому что он:

- самый тупой
- самый честный
- минимально фантазирует
- позволяет проверить сам принцип до всяких улучшений

Если даже он не живёт,
то дальше усложнять пока рано.

## Что именно реализуется

Минимальный объект:

- `1D ring`
- один `crazy-trace`
- `core + trace + phase`
- carry-state `A`
- seed-grown field
- fingerprint

## Состав состояния

У кольца есть:

1. `core[i]`
   Основное поле в `0..59048`

2. `trace[i]`
   Активный след

3. `phase[i]`
   Малый позиционный модификатор

4. глобальный `A`
   carry-state активного trace

5. текущая позиция `p`

## Seed

Seed задаёт:

- значение carry-state
- стартовое положение trace
- начальное наполнение нескольких узлов

Остальная среда разворачивается из seed через `crazy`.

## Tick baseline

На каждом тике:

1. берётся текущий узел `p`
2. берётся сосед `q = p + 1`
3. operand = `core[q]`
4. `res = crazy(A, operand)`
5. `A = res`
6. `core[p] = res`
7. `trace[p] = crazy(trace[p], res)`
8. `p = q`

Это и есть baseline `crazy-step`.

## Что логируется

Минимальные метрики:

- tick
- position
- carry `A`
- fingerprint
- trace_density
- distinct_core_count
- distinct_trace_count

## Что считаем признаком жизни

Baseline считается живым,
если:

- trace делает несколько полных кругов
- fingerprint не залипает сразу
- поле не схлопывается мгновенно в одно значение
- trace реально переписывается

## Что считаем смертью

Baseline считается мёртвым,
если:

- всё быстро становится одинаковым
- всё уходит в шум без повторяемого следа
- один полный круг не меняет поведение второго круга

## Реализация

Первая реализация делается на `Lua`.

Она должна быть:

- маленькой
- уродливой
- прозрачной
- пригодной для быстрого масштабирования кольца
