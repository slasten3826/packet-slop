# CPU/GPU PU Economy Split

Этот документ фиксирует следующий важный вывод:

экономика `PU` почти наверняка не будет общей для `CPU` и `GPU`.

## Коротко

Одна и та же формула `PU` для обеих веток начинает выглядеть фальшиво.

Причина простая:

- `CPU` и `GPU` по-разному переваривают саму жизнь слоя
- значит они по-разному тратят процессуальную стоимость

## Что уже видно

По `L1` уже было видно:

- `CPU` любит `wide sparse chaos`
- `GPU` любит `dense pressured chaos`

По первым `L2` и `L3` стендам это начало проявляться и в экономике:

- `CPU` тяготеет к более редким и дорогим актам
- `GPU` тяготеет к массовой ранней активации большого тела

То есть различие идёт не только по скорости исполнения, а по самой форме расхода.

## Что это значит для PU

Гипотеза:

- `CPU`-ветка может честно жить с моделью, где `L2 size` довольно хорошо описывает `PU_max`
- `GPU`-ветка требует более полевой модели, где размер слоя сам по себе не равен реальной цене жизни

То есть:

- для `CPU` геометрия и ёмкость слоя могут почти совпадать
- для `GPU` геометрия слоя и цена массовой активации расходятся

## CPU truth

На `CPU` правдоподобна такая логика:

- меньше активных узлов одновременно
- дороже индивидуальный акт
- выше цена за сложность маршрута
- `PU` ближе к стоимости точечных решений

Отсюда и гипотеза:

`L2 size -> PU_max`

Для `CPU` может оказаться близкой к правде.

## GPU truth

На `GPU` правдоподобнее другое:

- очень много узлов включаются сразу
- локальный акт сам по себе дешёв
- дорогим становится удержание большого поля во времени
- важна плотность одновременного вовлечения, а не только голый размер слоя

То есть для `GPU` одной формулы

`PU_max ~ L2_size`

недостаточно.

Нужно учитывать ещё:

- сколько узлов реально активны одновременно
- как долго держится массовая активация
- насколько плотен ранний разогрев слоя

## Следствие

Получается две разные экономики:

### CPU economy

- точечная
- узкая
- дорогая за акт
- более привязанная к геометрической ёмкости

### GPU economy

- полевая
- массовая
- дешёвая локально
- дорогая по удержанию большого живого поля

## Практический вывод

Это значит:

- нельзя насильно держать одну и ту же `PU`-формулу для обеих веток
- нужно проектировать `CPU`- и `GPU`-экономику как родственные, но разные тела одного закона

То есть общим остаётся:

- сам смысл `PU`
- его место в цикле
- общая логика эмиссии / переноса / сжигания / учёта

А расходная физика уже может разойтись по веткам.

## Самая короткая формула

- `CPU` тратит `PU` как стоимость редких решений
- `GPU` тратит `PU` как стоимость удержания массового поля

## Статус

Это уже не просто философская догадка.

Первые стенды показывают, что разница в экономике начинает проявляться так же явно, как раньше проявилась разница в самой физике исполнения.
